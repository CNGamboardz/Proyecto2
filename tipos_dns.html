Definición Resolución de Nombres Tipos de DNS

Tipos de DNS

Servidor de resolución de código auxiliar DNS:


Es un componente de software que normalmente se encuentra en hosts de terminales y que genera consultas de DNS cuando los programas de aplicación que se ejecutan en computadoras de escritorio o dispositivos móviles necesitan resolver nombres de dominio DNS.

Servidor de resolución recursiva DNS:


Un solucionador recursivo puede residir en un enrutador doméstico, estar alojado en un proveedor de servicios de Internet o ser proporcionado por un tercero, como, por ejemplo: Por ejemplo, el solucionador recursivo de DNS público de Google en 8.8.8.8 o el servicio DNS de Cloudflare en 1.1.1.1.

Servidor autoritario de DNS:


Un servicio DNS autorizado proporciona un mecanismo de actualización que los desarrolladores utilizan para administrar nombres DNS públicos.

De esta forma, responde a las consultas de DNS convirtiendo los nombres de dominio en direcciones IP para que las computadoras puedan comunicarse entre sí.



CLASIFICACIÓN DE DNS


Existe varios tipos de registros y se van clasificando para que sirven cada una de ellas, como, por ejemplo:




A:


Apunta a una dirección IP.

AAAA:


Apunta a una dirección IPv6

CNAME:


Reenvía un dominio o subdominio a otro dominio

MX:


Dirige el correo a un servidor de correo electrónico



TXT:


Permite que un administrador pueda almacenar notas de texto en el registro


NS:


Almacena el servidor de nombres para una entrada DNS

PTR:


Asocia una dirección IP con un nombre de dominio

SOA:


Contiene información sobre la zona DNS, como el servidor autoritativo primario



ESTRUCTURA Y JERARQUIA DEL DNS


Los DNS usan una jerarquía y una estructura para almacenar información que está asociada
con el DNS, como el internet para ello existe como funciona los servidores, la estructura
se puede ver de esta manera:



  • Tiene una estructura de árbol lógica, donde cada nodo o dominio tiene un nombre y puede contener subdominios.
  • Los dominios y subdominios se agrupan en zonas para permitir la gestión distribuida del espacio de nombres.
  • Identifica su posición en la jerarquía DNS lógica, separando cada rama del árbol con un punto.
    		•


    La jerarquía funciona con diferentes tipos de servidores DNS que se consultan de una forma recursiva para resolver cualquier petición


    Servidores raíz:


    Hay aproximadamente 400 servidores de nombres raíz repartidos por todo el mundo, administrados por 13 organizaciones diferentes. Proporcionan la dirección IP del servidor TLD.

    Servidor de dominio de nivel superior (TLD):


    DEl servidor TLD está disponible para todos los dominios de nivel superior como .com, .org, .net, .edu, .gov, etc.

    Proporcionan las direcciones IP de servidores de nombres autorizados para cada dominio.

    Servidores de nombres autorizados:


    También llamados servidores primarios o secundarios, estos servidores son responsables de un dominio específico y contienen los registros DNS para ese dominio.

    Servidor recursivo:


    Este es un servidor DNS que tiene la capacidad de reenviar solicitudes a otro servidor si ese servidor no tiene la dirección solicitada.
